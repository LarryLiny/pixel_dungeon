## 1. 像素美术质量

### 现状评价: 好（基础层面）

**优点:**
- 所有精灵严格遵守 16x16 基础尺寸规范，符合像素美术限制
- 每个精灵使用了 3-5 层颜色（暗轮廓 -> 主体填充 -> 高光 -> 核心亮点），符合艺术圣经的 4-step brightness ramp
- 敌人剪影差异化良好：史莱姆（圆弧）、蝙蝠（三角翅膀）、骷髅（细长）、兽人（宽大）、幽灵（椭圆透明）、恶魔（高+角）——黑底填色测试基本可识别
- 28 个物品图标全部采用程序化生成，风格一致
- `imageSmoothingEnabled = false` 确保像素锐利

**问题列表:**

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| P1 | 无精灵动画帧 | 高 | 所有角色、敌人均为单帧静态图。玩家无行走动画、敌人无移动/攻击动画。艺术圣经要求 "Expression Without Animation" 通过行为和眼睛表达，但核心移动应有至少 2 帧呼吸/弹跳效果 |
| P2 | 缺少 Boss 专属精灵 | 高 | `pixelAssets.ts` 中无 Boss 纹理生成函数。Boss 疑似复用 demon 纹理（scale 2.2），视觉上无法区分普通恶魔和 Boss |
| P3 | 融合产物图标缺失 | 高 | 艺术圣经定义了 20 个融合产物的 split-blend 构图规则（左半=父A，右半=父B），但 `pixelAssets.ts` 中无任何融合产物专用图标纹理。HUD 中融合物品 fallback 使用第一个输入的图标，无法体现融合关系 |
| P4 | 玩家精灵缺少辨识标记 | 中 | 艺术圣经要求 "Add a 1px white highlight on the helmet (top-left)" 和 "subtle sword/weapon indicator on the back"，当前均未实现 |
| P5 | 子弹精灵过于简单 | 低 | 基础子弹为 6x6 单一圆形，各武器类型仅靠 `setTint()` 区分。冰冻弹、毒蛇弹、死亡镰刀等特殊武器在飞行中视觉差异不足 |
| P6 | 怪物受击闪白时间过短 | 低 | `Enemy.takeDamage()` 中 `setTint(0xffffff)` 仅持续 80ms。快速战斗中可能被忽略 |

### 修改建议:

| 建议 | 优先级 | 工作量 |
|------|--------|--------|
| 为玩家添加 2 帧行走动画（腿部交替） | 高 | 小 |
| 为史莱姆添加 2 帧弹跳动画 | 高 | 小 |
| 创建 Boss 专属精灵（至少 24x24 基础） | 高 | 中 |
| 实现融合产物 split-blend 图标（20个） | 高 | 大 |
| 玩家头盔加 1px 白色高光 | 中 | 极小 |
| 受击闪白延长至 120ms | 低 | 极小 |

## 2. UI/HUD 设计

### 现状评价: 好（功能层面）

**优点:**
- HUD 严格使用 screen-space（`setScrollFactor(0)`），不与游戏世界冲突
- 字体尺寸动态缩放：`Math.min(sw/800, sh/600)` 比例，适配不同分辨率
- 技能栏底部居中，触控友好（slot 40x40，接近 Apple HIG 44x44 标准）
- 技能栏可动态扩展（融合增加技能时宽度增长）
- 分类颜色系统一致：武器=橙边、增益=青边、防御=绿边，背景色对应深色
- 消息系统堆叠显示，后 40% 时间渐隐，符合艺术圣经规范
- HP 条变化即时更新，无缓动（"delayed HP feels wrong in combat" -- 正确）

**问题列表:**

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| U1 | 无 HP 低血量视觉警告 | 高 | HP 条始终为 `#ff3344`，无论血量多少。艺术圣经要求通过颜色变化传达紧迫感。当 HP < 30% 时应有脉动/变深红效果 |
| U2 | HP 条尺寸过小 | 中 | `hpBarH = Math.round(14 * scale)`，在 800x600 基准下仅 14px。移动端可能仅 ~10px，难以点击或快速读取。建议最小 16px |
| U3 | 技能栏无选中/激活指示 | 中 | 技能槽只显示图标+等级，无视觉指示哪个技能正在生效（如旋转火球的运转状态） |
| U4 | 稀有度边框粗细无差异 | 中 | 艺术圣经要求稀有度通过边框粗细区分（Rare=2px, Epic=3px, Legendary=4px），当前所有技能槽统一 2px 边框 |
| U5 | 按钮无触摸反馈动画 | 低 | 所有按钮仅有颜色变化（pointerover/pointerout），无按下缩放效果。触屏设备缺少 pointerover，反馈仅靠 pointerdown |
| U6 | 菜单无背景装饰 | 中 | MenuScene 仅有纯色背景 `#1a1a2e` + 文字按钮。艺术圣经要求 "Warm gold focal light on dark void" 的氛围感，当前完全空白 |
| U7 | 字体未使用本地化系统 | 中 | CLAUDE.md 技术规则要求 "All UI text must go through the localization system -- no hardcoded user-facing strings"，但所有场景的中文文字均为硬编码字符串 |
| U8 | 游戏结束界面使用原生 HTML input | 低 | GameOverScene 中名字输入使用 DOM `<input>` 元素叠加在 Canvas 上，视觉风格与游戏内像素美术不统一 |

### 修改建议:

| 建议 | 优先级 | 工作量 |
|------|--------|--------|
| HP < 30% 时红色脉动 + 深红渐变 | 高 | 小 |
| HP 条最小高度提升至 16px | 中 | 极小 |
| 融合物品添加 3px/4px 边框（按稀有度） | 中 | 小 |
| 菜单背景添加暗纹装饰或光晕 | 中 | 小 |
| 按钮添加按下缩放（0.95x）效果 | 低 | 极小 |
| 技能激活时添加微弱脉动动画 | 低 | 小 |

## 3. 战斗反馈

### 现状评价: 好

**优点:**
- 完整的反馈层次体系：命中闪白 -> 伤害数字 -> 击杀慢动作 -> 死亡粒子 -> 屏幕震动 -> 相机闪光
- 伤害数字区分普通/暴击（字号 12 vs 16，颜色红 vs 金，缩放 1.0 vs 1.5）
- 击杀慢动作分级（heavy: 0.3x / 150ms, normal: 0.5x / 80ms），有防叠加逻辑
- 融合特效层次丰富：屏幕闪光 + 震动 + 三重扩展环（黄->橙->白）
- 死亡爆炸有环形扩展 + 粒子扩散双层效果
- `onKill` 慢动作仅在不在慢动作中时触发，避免堆叠

**问题列表:**

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| C1 | 击杀慢动作强度过高 | 中 | heavy kill 的 0.3x timeScale + 150ms 在高频击杀时可能导致节奏断裂。建议 heavy 0.4x / 120ms |
| C2 | 死亡粒子颜色单一 | 中 | `deathExplosion()` 默认使用 `0xffaa44`（橙色），不随敌人类型变化。史莱姆死亡应为绿色粒子，骷髅应为灰白色 |
| C3 | 缺少持续型状态指示 VFX | 高 | 毒素伤害、冰冻减速、无敌状态等无持续性视觉指示。艺术圣经要求 "Poison = drip animation + skull icon"，但代码中仅依赖 `setTint` 做受击闪白 |
| C4 | 伤害数字使用 DOM Text 而非位图字体 | 低 | `Phaser.GameObjects.Text` 在大量使用时有性能开销。200 发子弹场景下可能产生上百个伤害数字 |
| C5 | 闪避/无敌无特效 | 中 | 幽灵步（ghost_step）的 phase 行为仅通过 `setAlpha(0.3)` 表现，无发光或残影效果 |
| C6 | 拾取特效缺失 | 中 | 拾取物品仅有 levelUpFlash + levelUpRing，无物品吸入动画或弧线轨迹 |

### 修改建议:

| 建议 | 优先级 | 工作量 |
|------|--------|--------|
| 死亡粒子颜色传入敌人主色 | 中 | 小 |
| 为毒/冰/无敌添加持续 VFX 指示 | 高 | 中 |
| ghost_step 添加残影效果 | 中 | 中 |
| 拾取时物品缩放吸入动画 | 中 | 小 |
| 考虑伤害数字对象池 | 低 | 中 |

## 4. 场景美术

### 现状评价: 需要改进

**优点:**
- 地板纹理有石块分区和裂纹细节（32x32 tile 内含 4 块 + 3 条裂纹）
- 墙壁有清晰的砖石图案和砂浆线
- 地牢生成算法产生合理的房间+走廊布局
- 摄像头跟随平滑（lerp 0.08）

**问题列表:**

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| E1 | 无氛围照明/色调变化 | 高 | 艺术圣经要求 "The dungeon starts cold and dim (blue-tinted ambient)" 并随波次变暖。当前 `GameScene` 完全没有 `setBackgroundColor()` 或任何光照系统。背景默认黑色，无温度演化 |
| E2 | 无环境装饰物 | 高 | 艺术圣经指出 "Current state: Zero props. Recommended minimum: 2-3 decorative elements per room"，但实际确实为零装饰。房间空旷，只有地板+墙壁 |
| E3 | 缺少 Boss 区域特殊纹理 | 中 | 艺术圣经定义了 Boss Arena Floor（`#3d2228` 红色石块）和 Treasure Room Floor（金脉纹理），均未实现 |
| E4 | 地板纹理变化不足 | 低 | 每个房间使用完全相同的 floor 纹理，无深浅变化 |
| E5 | 无暗角/边缘效果 | 中 | 画面边缘无 vignette 或雾效果，远处的墙壁与脚下的地板无视觉区分 |
| E6 | 无走廊/房间视觉区分 | 低 | 艺术圣经定义了走廊使用更暗的 `#282840` 地板，但未实现 |

### 修改建议:

| 建议 | 优先级 | 工作量 |
|------|--------|--------|
| 添加相机背景色（冷蓝色 `#0d0d1a`） | 高 | 极小 |
| 随波次改变背景色温度 | 高 | 小 |
| 为房间添加 2-3 种装饰物（裂纹/碎石/苔藓） | 高 | 中 |
| 添加 vignette 效果（边缘变暗） | 中 | 小 |
| 实现 Boss 区域红色纹理 | 中 | 小 |

## 5. 粒子/VFX

### 现状评价: 需要改进

**优点:**
- 融合闪光效果精致（32x32 三层渐变 + 放射线 + 角线）
- 闪电链视觉清晰（`add.line` 连线 150ms 后销毁）
- 激光束三层渲染（外发光 -> 主体 -> 核心白线），光子炮有脉动冲击圈
- 毒云区域有淡入淡出（`0x44cc22, alpha 0.25`）
- 旋转火球有元素色变体（地狱火=红、太阳风暴=橙、追踪=深红）

**问题列表:**

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| V1 | 无碎片的稀有度脉动效果 | 中 | 艺术圣经要求 Uncommon+ 有脉动、Legendary 有粒子轨迹，但 Pickup 仅使用静态 glow ring |
| V2 | 拾取物无浮动动画 | 中 | Pickup 定义了 `bobOffset` 但 `update()` 方法中没有使用它做上下浮动。拾取物完全静止 |
| V3 | 升级特效仅屏幕闪光 | 低 | 拾取道具时只有 levelUpFlash（白色闪光）和 levelUpRing（扩展环），无道具特有的升级光效 |
| V4 | 缺少环境粒子 | 中 | 无漂浮灰尘、火花、环境光点等背景粒子，场景缺乏 "alive" 感 |
| V5 | 闪电使用 `add.line` 不够精致 | 低 | 闪电链仅用直线连接，无锯齿/分叉效果。pixelAssets 中有精致的闪电纹理但可能未在连锁中使用 |
| V6 | 冰冻波视觉不够突出 | 中 | 冰冻波使用 `add.arc` 半透明弧形，与战斗场景的对比度不足，容易在混乱中被忽略 |

### 修改建议:

| 建议 | 优先级 | 工作量 |
|------|--------|--------|
| 实现 Pickup 上下浮动动画 | 中 | 极小 |
| 碎片稀有度脉动（alpha 呼吸） | 中 | 小 |
| 添加 3-5 个背景漂浮粒子 | 中 | 小 |
| 冰冻波增加不透明度和边缘发光 | 中 | 极小 |
| 闪电链使用锯齿折线 | 低 | 小 |

## 6. 色彩系统

### 现状评价: 好

**优点:**
- 元素色彩家族与艺术圣经高度一致：Fire `#ff4422`, Ice `#88ccff`, Lightning `#4488ff`, Poison `#44cc22`, Dark `#aa44ff`, Holy `#ffcc44` -- 全部匹配
- 稀有度颜色系统正确：Common `0xcccccc`, Uncommon `0x44cc44`, Rare `0x4488ff`, Epic `0xaa44ff`, Legendary `0xffaa00`
- 物品分类颜色与圣经完全匹配：Weapon `0xff8844`/`0x663311`, Augment `0x44eeff`/`0x113355`, Defense `0x44ff88`/`0x115533`
- 4-step brightness ramp 在精灵中广泛使用
- 深海军蓝背景 `#1a1a2e` / `#0a0a1a` 贯穿所有场景

**问题列表:**

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| K1 | 色盲安全措施未实现 | 高 | 艺术圣经定义了 4 种色盲混淆场景的 Backup Cue，但代码中无实现：无稀有度文字标签（"R"/"E"/"L"）、无稀有度边框粗细差异、Fire/Poison 无动画差异 |
| K2 | 融合产物 color 字段未与元素家族对齐 | 中 | 部分融合产物的 `color` 字段（如 `freeze_shotgun: '#88ccff'`）使用了与父元素不同的中间色，但在视觉反馈（伤害数字、拾取闪光）中可能产生混淆 |
| K3 | 无 grayscale 可读性验证 | 中 | 艺术圣经要求 "Convert a screenshot to grayscale -- can you still identify your character, the enemies, and the dropped items?"，但游戏无灰度测试模式 |
| K4 | 伤害数字颜色不随元素变化 | 低 | `showDamage()` 默认使用 `#ff4444`，不随武器元素变化。火系伤害应为橙色，冰系应为蓝色等 |
| K5 | 拾取物 glow ring 颜色单一 | 低 | 拾取物使用分类颜色的 glow ring（`alpha 0.25`），但无脉动效果。高稀有度拾取物应有更亮/脉动的光晕 |

### 修改建议:

| 建议 | 优先级 | 工作量 |
|------|--------|--------|
| 实现稀有度边框粗细差异化 | 高 | 小 |
| 添加稀有度文字标签（至少 Legendary 显示 "L"） | 高 | 小 |
| Fire 添加闪烁动画 / Poison 添加滴落效果 | 高 | 中 |
| 伤害数字颜色传入元素色 | 低 | 小 |
| Legendary 拾取物 glow ring 脉动 | 中 | 小 |

## 7. 艺术圣经合规性

### 现状评价: 部分合规

逐项核查艺术圣经规范：

| 规范 | 合规 | 说明 |
|------|------|------|
| **剪影可识别性** | 是 | 7 种敌人剪影差异化良好 |
| **发光即信息** | 部分 | 拾取物有 glow ring，但无元素发光强度层级（Rarity glow intensity） |
| **亮度层级优先** | 是 | 精灵使用 4-step ramp，背景不超过 Body 亮度 |
| **氛围情绪表** | 否 | 无波次温度演化，无 Boss 红色偏移，无 Game Over 去饱和 |
| **形状语言** | 是 | 玩家紧凑对称、敌人各具特征、UI 扁平化 |
| **主色调系统** | 是 | 7 种主色全部匹配 |
| **元素色家族** | 是 | 6 种元素 4-step ramp 一致 |
| **稀有度颜色** | 是 | 5 级稀有度颜色匹配 |
| **分类颜色** | 是 | 3 种分类颜色+背景完全匹配 |
| **色盲安全** | 否 | 无 Backup Cue 实现 |
| **波次温度演化** | 否 | 无随波次变化的色彩方案 |
| **玩家视觉原型** | 部分 | 颜色正确，但缺少头盔高光和背部武器指示 |
| **融合产物 split-blend** | 否 | 完全未实现 |
| **地牢建筑风格** | 部分 | 砖石+砂浆正确，但无装饰物、无特殊房间纹理 |
| **HUD 架构** | 是 | Screen-space only，monospace 字体 |
| **字体层级** | 是 | 尺寸和颜色基本匹配圣经表格 |
| **技能栏设计** | 部分 | 尺寸和颜色匹配，但缺少稀有度边框粗细差异 |
| **UI 动画规则** | 部分 | HP 即时更新（正确），融合闪光 300ms（正确），但技能栏扩展无 200ms 过渡 |
| **移动端适配** | 是 | 触摸区域、摇杆区域、安全区域处理正确 |
| **资源命名规范** | 是 | 纹理键命名符合规则 |
| **精灵尺寸层级** | 是 | Micro/Small/Standard/Tile/Effect 层级均有对应 |
| **性能预算** | 是 | 子弹 80/200、敌人 25/50、拾取 15/20 均在预算内 |

**合规率**: 约 70%（16/23 项完全合规）

### 关键缺失项总结:

1. **融合产物视觉系统** -- 20 个融合图标 + split-blend 构图完全缺失
2. **波次温度演化** -- 无任何随波次变化的视觉变化
3. **色盲安全措施** -- 无备份提示符实现
4. **环境装饰** -- 零装饰物
5. **Boss 专属视觉** -- 无独立 Boss 精灵或区域纹理

## 优先级排序总览

### 高优先级（影响核心体验）

| # | 建议 | 类别 | 预估工作量 |
