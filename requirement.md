## 第 1 章：产品概述

### 1.1 产品描述

「地牢肉鸽」是一款类 Vampire Survivors 的俯视角自动射击 Roguelike 游戏。玩家在地牢中自动攻击涌来的敌人，通过拾取道具、升级技能、融合装备来强化角色，在波次推进中尽可能存活更久。游戏支持移动端触摸操作，具备持久化升级、碎片解锁、每日挑战和排行榜等元游戏系统。

### 1.2 目标用户

- 休闲游戏玩家，偏好 Roguelike/Survivor 类游戏
- 移动端为主，支持桌面端
- 中文用户（UI 全中文）

### 3.2 核心战斗 (GameScene)

#### 3.2.1 用户故事

> 作为一个玩家，我希望操控角色在地牢中自动攻击敌人，拾取道具强化自己，在越来越难的波次中尽可能存活。

#### 3.2.2 游戏世界

- **地图**：60×45 格的地牢网格，每格 32 像素，总尺寸 1920×1440
- **地牢生成**：随机生成 12-18 个矩形房间，用 2 格宽的 L 形走廊连接，中心 7×7 区域为玩家出生点
- **相机**：跟随玩家，有平滑插值（lerp 0.08）

#### 3.2.3 游戏循环状态机

```
┌──────────┐  开始游戏  ┌──────────┐
│  初始化   │ ────────→ │  战斗中   │
│ create() │           │ update() │
└──────────┘           └────┬─────┘
                            │
                   玩家 HP <= 0
                            │
                            ↓
                       ┌──────────┐  1秒延迟  ┌──────────────┐
                       │  死亡     │ ────────→ │ GameOverScene │
                       │ onDeath  │           │              │
                       └──────────┘           └──────────────┘
```

#### 3.2.4 每帧更新流程

```
update(time, delta)
  │
  ├── 累计游戏时间 / 定时清理（每2秒）
  │
  ├── 读取输入 → 移动玩家
  │
  ├── 波次系统更新 → 推进波次（每30秒）
  │
  ├── 敌人生成 → 按波次配置间隔 + 紧急生成（<3只时）
  │
  ├── 拾取物生成 → 每8秒
  │
  ├── 融合检查 → 每2秒自动融合
  │
  ├── 敌人 AI 更新 → 所有存活敌人
  │
  ├── 自动射击系统 → 瞄准最近敌人
  │
  ├── 被动效果 → 排斥光环 / 护盾球 / 磁铁吸引
  │
  ├── HUD 更新
  │
  ├── 战斗反馈更新（慢动作等）
  │
  └── 死亡检测 → HP <= 0 → 跳转结算
```

#### 3.2.7 道具系统

道具分为三类：

| 分类 | 数量 | 说明 | 叠加规则 |
|------|------|------|----------|
| 武器 weapon | 10种 | 决定攻击形态 | 同类武器替换，不叠加 |
| 增益 augment | 9种 | 强化武器属性 | 同类可叠加至等级9 |
| 防御 defense | 9种 | 保护玩家 | 同类可叠加至等级9 |

**拾取规则**：
- 击杀敌人 25% 概率掉落道具，10% 概率掉落药水
- 道具拾取后自动装备到技能槽
- 最多 5 个技能槽（融合可额外增加）
- 拾取同类道具升级而非新增（叠加至最多等级 9）

#### 3.2.8 融合系统

当玩家背包中存在满足条件的两个道具时，自动触发融合（每 2 秒检查一次）：

**融合规则**：
- 两个道具必须在融合配方中配对
- 两个道具等级均 >= 配方要求的最低等级（通常为 3 级）
- 融合消耗两个道具，产出 1 个新的融合道具
- 融合后释放一个技能槽位

### 3.7 每日挑战系统

#### 3.7.1 用户故事

> 作为一个玩家，我希望每天有不同的挑战目标可以完成，增加游戏可玩性。

#### 3.7.2 业务规则

- 每日生成 3 个挑战（简单/中等/困难各 1 个）
- 使用基于日期的种子随机数，所有玩家同一天看到相同挑战
- 挑战类型：

| 类型 | 说明 |
|------|------|
| kill_with_weapon | 用指定武器击杀一定数量敌人 |
| survive_wave | 存活到指定波次 |
| no_damage_taken | 前N波不受伤 |
| kill_enemies | 击杀指定数量敌人 |
| collect_items | 拾取指定数量道具 |
| no_potion | 前N波不使用药水 |
| defense_only | 仅装备防御道具存活到指定波次 |

- 简单挑战 1 个目标，中等 2 个，困难 3 个
- 30 个挑战模板（各难度 10 个）循环使用

## 第 4 章：业务规则

### 4.1 计算规则

| 计算项 | 输入 | 公式/逻辑 | 输出 |
|--------|------|-----------|------|
| 每波敌人数 | 波次号 N | 5 + N × 2 | 敌人数量 |
| 刷怪间隔 | 波次号 N | max(500ms, 2000 - N × 100) | 毫秒间隔 |
| 敌人速度倍率 | 波次号 N | 1 + N × 0.05 | 速度倍率 |
| 敌人血量倍率 | 波次号 N | 1 + N × 0.10 | 血量倍率 |
| 掉落加成 | 波次号 N | N × 0.02 | 额外掉落率 |
| 实际射击间隔 | 基础间隔 400ms × 攻速倍率 | 400 × attackSpeedMul | 毫秒间隔 |
| 实际伤害 | 基础伤害 × 伤害倍率 × 武器等级值 | BULLET_DAMAGE × damage × levelValue | 最终伤害 |
| 实际受伤 | 敌人伤害 × (1 - 减伤比例) | enemy.damage × (1 - shieldReduction) | 扣血量 |
| 金币获取 | 击杀数 + 波次数 | 累计敌人分数值 + 波次奖励 | 金币数 |

### 4.2 条件逻辑

| 规则名称 | 条件描述 | 效果 | 适用场景 |
|----------|---------|------|----------|
| 紧急刷怪 | 存活敌人 < 3 | 立即生成新敌人 | 战斗循环 |
| 道具掉落 | 击杀敌人时 random < 25% | 掉落道具拾取物 | 击杀事件 |
| 药水掉落 | 击杀敌人时 random < 10% | 掉落药水拾取物 | 击杀事件 |
| 碎片掉落 | 击杀敌人时，波次在碎片掉落范围内，random < 掉落率 | 掉落对应碎片 | 击杀事件 |
| 自动融合 | 每 2 秒检测背包中满足配方的道具对 | 消耗两个道具，产出融合道具 | 融合系统 |
| 道具升级 | 拾取已装备的同类道具 | 等级+1（最高9级） | 拾取系统 |
| 道具新增 | 拾取未装备的道具且有空槽位 | 新装备道具 | 拾取系统 |
| 无敌帧 | 受伤后短暂无敌 | 一定时间内不再受伤 | 受伤事件 |
| 排斥光环 | 装备排斥光环时 | 持续推开范围内敌人并伤害 | 被动效果 |
| 护盾球 | 装备旋转护盾时 | 环绕球体碰撞伤害敌人 | 被动效果 |
| 磁铁吸引 | 装备磁力场时 | 远处拾取物被吸引向玩家 | 被动效果 |
| 治愈披风 | 装备治愈披风时 | 每秒回复一定生命值 | 被动效果 |
| 幽灵步 | 装备幽灵步时 | 每 8 秒触发 1 秒无敌 | 被动效果 |
| 圣光守护 | 装备圣光守护且血量低于阈值 | 触发大回血 | 被动效果 |
| 死亡镰刀 | 装备死亡镰刀且敌人血量低于斩杀线 | 直接秒杀 | 攻击判定 |

### 4.3 数据处理规则

| 操作 | 规则描述 |
|------|----------|
| 排行榜排序 | 按分数降序，最多保留 20 条 |
| 道具稀有度颜色 | 普通=灰、优秀=绿、稀有=蓝、史诗=紫、传说=金 |
| 碎片稀有度颜色 | 同道具稀有度颜色 |
| 分数计算 | 每次击杀累加敌人的分数值 |
| 金币计算 | 基于击杀数和波次的综合计算 |
